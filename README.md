# (Reversi) Othello

Реализована мультиплеерная версия игры Othello. Архитектура приближена к микросервисной.

## Технологический стек

*   **Backend:** Java 21, Quarkus, Vert.x
*   **База данных:** MongoDB
*   **Инфраструктура:** Docker, Docker Compose, Prometheus
*   **Кеш** Redis
*   **Безопасность** JWT
*   **Десктоп** JavaFX

## Архитектура и ключевые возможности

Смысл в том, что пользователь заходит в десктопное приложение, используя JavaFX. Затем у него есть возможность регистрации или входа. Также если он этого делать не хочет, он может поиграть сам с собой. Как асинхронный веб-клиент я использую Vert.x. В первую очередь никак не взаимодействует с БД сам, все запросы отсылаются на вершины Vert.x, кроме запросов об авторизации, которые находятся в отношении master-slave. Есть ParentVeticle(они смотрят за подчиненными вершинами, использя AsyncMap, и лишь они в праве рукводствуясь установленным ttl модифицировать состав подчиненных вершин), которые есть продюсеры ChildVerticle(которые ответственные за игровые сессии). Клиент обращается к ChildVerticle, чтобы получить информацию о доступных играх. В каждом лобби, в соответствиями с правилами игры, может быть лишь 2 человека. Человеку доступны правила игры по ссылке, если тот войдет в систему. Есть одно лишь маленькое но, что DTO пользователя не скрывает данные по типу пароля.
